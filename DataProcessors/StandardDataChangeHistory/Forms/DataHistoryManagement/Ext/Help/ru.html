Управление историей данных

Диалог открывается с помощью команды Еще - Управление историей данных.

Будет обработана: - строка содержит информацию о данных, для которых изменяется история данных.

Обновить историю данных - вызов обновления истории данных. С помощью флажков Выполнять обработку после записи версий и Выполнять удаление из обработки после записи версий можно явно указать необходимость выполнения ообработку после записи версий и удаление из очереди обработки после записи версий.

Выполнить обработку после записи версий - вызов обработка после записи версий без изменения состояния очереди обработки после записи версий.

Удалить из обработки после записи версии все весии, созданные ранее даты - позволяет очистить очередь обработки после записи версий. В качестве границы удаления подставляется граница по умолчанию - текущая дата минус 20 дней.
